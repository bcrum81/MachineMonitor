Loading…
Webhooks fire on cycle events (cycle.started, cycle.completed) and/or alarm events (alarm.<CODE>, e.g. alarm.IO0518) for the machines you select. Every request includes an X-CNC-Signature header (HMAC-SHA256 of the body using the subscription's secret) so the receiver can verify authenticity. Failed deliveries retry twice (2s, then 8s) before giving up.

Column Format
ColumnValueExample
A — Date Event timestamp, ISO-8601 UTC 2026-04-16T18:47:18.247596+00:00
B — Machine Machine display name (not ID) Brother Mill 5
C — Pallet Pallet number (1 or 2) 1
D — Program Program number at time of event 1260
E — Operation Start or Complete Complete
F — Total Time (seconds) Cycle duration in seconds — blank on Start rows, filled on Complete573.8
Duration is paired in memory: each cycle.completed is matched to the last cycle.started on the same (machine, pallet). After a service restart, the first completion per pallet will have a blank duration because no matching start was observed.
Writes are queued and run in a background thread — they never block polling or webhook delivery. If Sheets is temporarily unreachable, rows wait in memory and retry on the next poll. Queue depth is shown above.
